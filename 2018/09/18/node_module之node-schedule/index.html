译自 https://www.npmjs.com/package/node-schedule

Node Schedule 是一个灵活的，类似于 cron 又不完全类似于 cron 的，在 Node.js 中使用的任务调度器。它允许你在特定日期，使用可选的循环规则来安排任务。它只在任何给定时间使用单个计时器(而不是每秒钟/分钟重新评估即将要做的任务)。

概述

Node Schedule 是基于时间点的排程，而不是基于时间段的排程。虽然你可以很容易地按照自己的意愿进行转换，但是如果你只想”每5分钟运行一次某个函数”，你会发现 setInterval 更容易使用，也会更合适。但是如果你想”在每个月的第三个星期二的每个小时的 20 分和 50 分运行一次某个函数”，你会发现 Node Schedule 更符合你的需要。另外，与真正的 cron 不同，Node Schedule 支持 Windows，因为现在完全支持 node 运行时。

注意 Node Schedule 是为进程内的排程而设计的，也就是说，只要你的脚本正在运行，被安排的任务就会触发，执行完成时排程就会消失。如果你需要安排即使脚本没有在运行也会持续的任务，请考虑使用实际的 cron。

任务和排程

Node Schedule 中的每个被安排的任务都用一个 Job 对象来表示。你可以手动创建任务，然后执行 schedule() 函数来应用一个排程，或者使用下面所示的更方便的函数：scheduleJob()。

Job 对象是属于 EventEmitter 的，并在每次执行之后发出一个 scheduled 事件。他们还在每次计划运行时发出一个计划事件，在执行调用之前取消时发出一个 canceled 事件（两个事件都接收同一个 JavaScript date 对象作为参数）。注意任务第一次是立即被安排的，所以如果你是使用 scheduleJob() 这个函数来创建的话，你将错过第一个被安排的事件。但是你可以手动查询调用。

不支持的 Cron 特性

Currently, W (nearest weekday), L (last day of month/week), and # (nth weekday of the month) are not supported. Most other features supported by popular cron implementations should work just fine.
目前不支持 W（最近的工作日），L（某月/某周的最后一天）和 #（某月的第 n 个工作日）。受欢迎的 cron 实现支持的大多数其他特性应该可以更好的工作。

处理任务和任务的调用

有一些功能可以获取作业的信息，并处理任务及其调用。

job.cancel(reshedule)

你可以使用 cancel() 方法使任何任务无效:

所有任务的调用都将被取消。当你将参数 reschedule 设置为 true 时，任务将在之后重新调度。

job.cancelNext(reshedule)

此方法将使下一个计划或任务调用无效。当你将参数 reschedule 设置为 true 时，任务将在之后重新调度。

job.reschedule(spec)

该方法取消所有挂起调用，并使用给定的规范重新注册作业。成功/失败分别返回 true/false。

job.nextInvocation()

此方法返回此任务计划的下一次调用的日期对象。如果没有计划调用，该方法返回 null。

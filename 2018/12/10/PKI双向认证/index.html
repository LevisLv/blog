PKI双向认证
发表于 2018-12-10|Android
字数总计:851|阅读时长: 4 分钟
| 阅读量:|评论数:

一、单向认证

1、生成默认jks格式的CA秘钥库(root.jks)

keytool -genkeypair -alias root -keystore root.jks -storepass 123456 -keyalg RSA -keysize 2048 -validity 3650

2、从CA秘钥库(root.jks)导出CA证书(root.cer)

keytool -export -alias root -file root.cer -keystore root.jks -storepass 123456

3、生成p12格式的服务端秘钥库(server.p12)

keytool -genkeypair -alias server -storetype pkcs12 -keystore server.p12 -storepass 123456 -keyalg RSA -keysize 2048 -validity 3650

4、从服务端秘钥库(server.p12)导出服务端证书(server.cer)

keytool -export -alias server -file server.cer -keystore server.p12 -storepass 123456

5、根据服务端证书(server.cer)生成服务端证书请求文件(server.csr)

keytool -certreq -alias server -file server.csr -keystore server.p12 -storepass 123456

6、通过CA证书重签服务端证书，得到重签的服务端证书(server_by_ca.cer)

keytool -gencert -alias root -infile server.csr -outfile server_by_ca.cer -keystore root.jks -storepass 123456

7、将CA证书导入服务端秘钥库中，使其信任CA根证书

note: 这一步不能省，否则出现 keytool 错误: java.lang.Exception: 无法从回复中建立链接

keytool -import -alias root -file root.cer -keystore server.p12 -storepass 123456

8、将重签的服务端证书覆盖原来的

keytool -import -alias server -file server_by_ca.cer -keystore server.p12 -storepass 123456

9、服务端配置服务端秘钥库(server.p12)

10、客户端信任秘钥库导入签发服务端证书的CA证书(root.cer)

note: 在Android中可将其import进client_trust.jks中，使用KeyStore Explorer修改秘钥库类型(client_trust.jks -> client_trust.bks)，按如下方法使用：

keytool -import -alias root -file root.cer -keystore client_trust.jks -storepass 123456

二、双向认证

11、生成默认jks格式的客户端秘钥库(client.jks)

keytool -genkeypair -alias client -keystore client.jks -storepass 123456 -keyalg RSA -keysize 2048 -validity 3650

12、从客户端秘钥库(client.jks)导出客户端证书(client.cer)

keytool -export -alias client -file client.cer -keystore client.jks -storepass 123456

13、根据客户端证书(client.cer)生成客户端证书请求文件(client.csr)

keytool -certreq -alias client -file client.csr -keystore client.jks -storepass 123456

14、通过CA证书重签客户端证书，得到重签的客户端证书(client_by_ca.cer)

keytool -gencert -alias root -infile client.csr -outfile client_by_ca.cer -keystore root.jks -storepass 123456

15、将CA证书导入客户端秘钥库中，使其信任CA根证书

note: 这一步不能省，否则出现 keytool 错误: java.lang.Exception: 无法从回复中建立链接

keytool -import -alias root -file root.cer -keystore client.jks -storepass 123456

16、将重签的客户端证书覆盖原来的

keytool -import -alias client -file client_by_ca.cer -keystore client.jks -storepass 123456

17、服务端信任秘钥库导入客户端的CA证书

keytool -import -alias root -file root.cer -keystore server_trust.p12 -storepass 123456

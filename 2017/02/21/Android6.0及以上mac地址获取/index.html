Android6.0及以上mac地址获取
2017-02-21|Android
字数总计: 575|阅读时长: 2 分钟|阅读量:

引言

由于Android6.0及以上对权限的使用更加严格，在不动态申请某些权限的时候，获取mac地址变的复杂了起来。
目前获取mac地址的方法有以下几种：
方法一：传统的通过WifiManager来获取（需要申明ACCESS_WIFI_STATE权限）；
方法二：通过NetworkInterface.getNetworkInterfaces()获取（需要申明INTERNET权限）；
方法三：通过读取 /sys/class/net/wlan0/address来获取（不需申明权限）。

三种情况

根据两种权限的有无进行调研：

情况1

当没有申明ACCESS_WIFI_STATE权限时，方法一会抛出 java.lang.SecurityException: WifiService: Neither user XXX nor current process has android.permission.ACCESS_WIFI_STATE 异常，当申明了此权限时，能获取 02:00:00:00:00:00 地址（无效）；

情况2

当没有申明INTERNET权限时，方法二会抛出 java.net.SocketException: Permission denied 异常，当申明了此权限时，mac地址能获取成功；

情况3

经部分机型测试，通过adb shell查看均权限不够，在HUAWEI NXT-AL10 7.0和三星 Nexus 6P 7.1.1手机上，获取均为空字符串，但在ONEPLUS A3000 7.0手机上，方法三mac地址获取成功。具体情况不明。

#总结
综上所述：7.0及以上Android手机通过在Manifest中申明INTERNET权限，使用方法二能成功获取到mac地址。
注：目前只测试了这三款7.0及以上手机，大家也可以多试试其他机型，以上情况3问题暂不明，需进一步确认，如有问题请反馈。
